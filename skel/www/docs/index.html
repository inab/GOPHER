XCESC (XML Continuous Evaluation System Core) technical documentation

As any CES (continuous evaluation system), XCESC always follows the same conceptual steps:

  • Periodically gather representative queries for the methods and servers to assess which are novel (not previously used in previous submissions to the participant methods and servers).
  • Management of participants and evaluators in the assessment: registration, updates, programmed shutdowns, results gathering, etc...
  • Filtering and tidying up of the representative query sets used to assess the participants, based on some criteria. And then, the queries are sent to the active participants.
  • Sending of an augmented version of the representative query sets to the evaluators (assessment methods and servers), so the complexity level of the assessment is measured.
  • Statistics generation.
  • Give end user access to the gathered information which is public.

An example of an assessment participant service is also available (click sampleGOPHERService.pl to download it). Although it is a simple CGI written in Perl it is easy to translate it to other popular server languages, like a Java Servlet, a PHP page, Ruby On Rails, Python, etc... Also, it is only a template, because many services require too many resources and too much time to be run online. Developers will be able to write custom versions of the template to integrate with their internal services infrastructure, such as running the jobs in separate machines, putting the jobs in a software and jobs queue system, putting the jobs in a grid environment, ...
